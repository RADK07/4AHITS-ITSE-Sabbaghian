## Vorbereitung der Übungsumgebung
Um die heutigen Übungen durchführen zu können, muss man ein Script mit folgenden Inhalt erstellen und ausführen:

```sh
#!/bin/bash

# Gruppen anlegen
groupadd music
groupadd wrestle
groupadd emperors

# User anlegen und Gruppen zuweisen
useradd -m -G music elvis
useradd -m -G music,wrestle blondie
useradd -m -G wrestle hogan
useradd -m -G emperors nero

# Passwörter auf 'kali' setzen
echo "elvis:kali" | chpasswd
echo "blondie:kali" | chpasswd
echo "hogan:kali" | chpasswd
echo "nero:kali" | chpasswd

# Lab-Struktur vorbereiten
mkdir -p /tmp/lab/gigs /tmp/lab/drafts
echo "Draft" > /tmp/lab/drafts/lyrics.txt
echo "Plans" > /tmp/lab/wrestle_plans.txt
chown -R root:root /tmp/lab
chmod -R 755 /tmp/lab
```

Im Script werden neue User mit Nutzernamen und Passwort erstellt und verschiedenen Gruppen zugewiesen.
Danach erstellt es einen Ordenr mit zwei Text-Files in denen Inhalt eingefügt wird.

## Übung (Analyse und Symbolic Mode)
### 1. Rechte auslesen
Zuerst soll man ermitteln, wer der Group-Owner und wer der User-Owner der Datei "/tmp/lab/drafts/wrestle_plans.txt ist.

**Befehl und Ausgabe:**
```sh
$ ls -l /tmp/lab/wrestle_plans.txt
-rwxr-xr-x 1 root root 6 Apr 27 05:58 /tmp/lab/wrestle_plans.txt
```

Hier sieht das "root" der Group-Owner und ebenfalls der User-Owner ist.
Die Permissions für den Group-Owner sind "read", "write" und "execute", für Mitglieder der Gruppe "execute" und "read" und für andere nur "execute".

### 2. Private Files
Nun soll man das File "notes.txt" erstellen, und mit chmod die Rechte so verändern, dass der other-Klasse das Read-Recht entzogen wird.

**Befehl:**
```sh
touch notes.txt
chmod o-r notes.txt
```
Wenn man nun versucht den Inhalt des Files auszugeben, ist das nicht möglich
```sh
$ cat notes.txt                 
cat: notes.txt: Permission denied
```

### 3. Group Sharing
Als nächstes soll der User alle Rechte, die Gruppe g die "read" und "write" Berechtigung und die other-Klasse keine Berechtigung über notes.txt haben dürfen

**Befehl:**
```sh
$ chmod u=rwx,g=rw,o= notes.txt
```

### 4. Skript-Vorbereitung
Hier soll man herausfinden, welcher Befehl benutzt wird, um für ein Script "myscript.sh" dem User und der Gruppe g das Recht gegeben wird, um dieses auszuführen.

**Befehl**
```sh
chmod ug+x myscript.sh
```

### 5. Kollektive Änderung
Als letztes soll der Gruppe g und der Klasse o das "write" und "execute" Recht für die Datei "data.txt" entzogen werden

**Befehl:**
```sh
chmod go-wx data.txt
```

## Übung (Ownership und Gruppenkollaboration)
### 1. Group-Change
Hier muss man auf den User "Blondie" zu den "sudoers" huínzufügen, dann zu diesem wechseln und die Datei "song.txt" erstellen und in "/tmp/lab/" kopieren.
Der Group-Owner soll auf Music gesetzt werden.

**Befehl:**
```sh
$ sudo usermod -aG sudo blondie
$ su - blondie
$ touch song.txt
$ sudo cp song.txt /tmp/lab
$ sudo chgrp music /tmp/lab/song.txt
```

### 2. Vergleich
Nun soll man herausfinden, wo der Unterschied beim Userwechsel mit "su" und mit "su -" liegt.

|Befehl|Bedeutung|
|------|---------|
|su |Userwechsel mit Beibehaltung der aktuellen Umgebung (Variablen des Verzeichnisses werden beibehalten)|
|su -|Userwechsel mit Laden dessen kompletter Umgebung (Home-Verzeichnis, Pfade, Aliase)|

### 3. User-Transfer
Als nächstes soll der User "elvis" zum User-Owner von "song.txt" machen

**Befehl:**
```sh
$ sudo chown elvis /tmp/lab/song.txt
```

### 4. Wrestler-Szenario
Nun soll man als "hogan" das File "/tmp/lab/training.txt" erstellen, den Group-Owner zu "wrestle" ändern,
die Berechtigungen von Group g auf "read" und "write" setzen und der o alle Rechte entziehen.

**Befehl:**
```sh
$ sudo usermod -aG sudo hogan
$ su - hogan
$ sudo touch /tmp/lab/training.txt
$ sudo chgrp wrestle /tmp/lab/training.txt
$ sudo chmod g+rw,o= /tmp/lab/training.txt
```

### 5. Einschränkungen
Zum Schluss soll man sich mit "elvis" einloggen und den Group-Owner zu "emperors" ändern.

**Befehl:**
```sh
su - elvis
sudo chgrp emperors /tmp/lab/lyrics.txt
```

Der Grund das der Befehl fehlschlägt ist, dass "elvis" kein Teil der "emperors" Gruppe ist.
