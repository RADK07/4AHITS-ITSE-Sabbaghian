## Übung (port probing)
Die Aufgabe ist mit "telnet", "netcat" und "curl" auf der Kali-Linux-VM portprobing zu betreiben. 
Dabei soll man herausfinden, ob ein ssh Server auf der Metasploitable-VM läuft 
und wie die Bezeichnung und die Versionsnummer des Webservers lautet.

**Portprobing mit Telnet:**
```sh
$ telnet 192.168.13.52 22                         
Trying 192.168.13.52...
Connected to 192.168.13.52.
Escape character is '^]'.
SSH-2.0-OpenSSH_4.7p1 Debian-8ubuntu1
```

**Portprobing mit netcad:**
```sh
$ nc 192.168.13.52 22 
SSH-2.0-OpenSSH_4.7p1 Debian-8ubuntu1
```

**Portprobing mit curl:**
```sh
$ curl -v telnet://192.168.13.52:22 
*   Trying 192.168.13.52:22...
* Connected to 192.168.13.52 (192.168.13.52) port 22
SSH-2.0-OpenSSH_4.7p1 Debian-8ubuntu1
```

## Übung (netcad port scan)
Nun soll man mit Netcad auf Kali-Linux überprüfen, welche Ports auf Metasploitable zwischen 1 und 30 laufen.

**Befehl:**
```sh
$ nc -zv 192.168.13.52 1-30
192.168.13.52: inverse host lookup failed: Unknown host
(UNKNOWN) [192.168.13.52] 25 (smtp) open
(UNKNOWN) [192.168.13.52] 23 (telnet) open
(UNKNOWN) [192.168.13.52] 22 (ssh) open
(UNKNOWN) [192.168.13.52] 21 (ftp) open
```

| Netcad-Option | Bedeutung | 
|--------------|--------------|
| -z | Baut Verbindung auf, jedoch ohne selbst Daten zu senden | 
| -v | Dient zur detaillierteren Ausgabe |

### Experimentieren mit Nmap Optionen
**1.**  Bei der -p- Optione werden mehr Ports gescant wie beim Default. Somit dauert der Scan länger.


**Befehl:**
```sh
$ nmap -p- 192.168.13.52
```

**2.** Da manche Netzwerke das Ping-Protokoll "ICMP" blockieren kann man mit der -Pn Option den Scan trotzdem ausführen.

**Befehl:**
```sh
$ nmap -Pn 192.168.13.52
```

**3.** Durch -sV wird ebenfalls die Version des Services angegeben

**Befehl:**
```sh
$ nmap -sV 192.168.13.52
!
!
PORT     STATE SERVICE     VERSION
21/tcp   open  ftp         vsftpd 2.3.4
22/tcp   open  ssh         OpenSSH 4.7p1 Debian 8ubuntu1 (protocol 2.0)
!
!
```

**4.** Mit der -O kann man das Operating System (OS) herausfinden

**Befehl:**
```sh
$ nmap -O 192.168.13.52
!
!
Device type: general purpose
Running: Linux 2.6.X
OS CPE: cpe:/o:linux:linux_kernel:2.6
OS details: Linux 2.6.9 - 2.6.33
Network Distance: 1 hop
 
OS detection performed. Please report any incorrect results at https://nmap.org/submit/ .
Nmap done: 1 IP address (1 host up) scanned in 1.59 seconds
```

**5.** Die Option -A bietet einen genaueren Scan durch kombinieren von verschiedenen Optionen 

**Befehl:**
```sh
$ nmap -A 192.168.13.52
```

## Übung (nmap cheat sheet)
Hierbei soll man jene verwendeten, sowie andere bedeutenden Nmap-Optionen auflisten

| Nmap-Option | Bedeutung | 
|--------------|--------------|
| -sn | Überprüft nur ob Hosts online sind und nicht die Ports |
| -sS | Syn-Scan ist ein Halboffener TCP-Scan | 
| -sT | Vollständiger TCP-Scan | 
| -sU | UDP-Scan welcher offene UDP-Ports erkennt wie zum Beispiel DNS, SNMP, DHCP, usw.| 
| -sV | Zeigt welcher Service unter der Portnummer läüft | 
| -p | Dient um spezifische Ports zu scannen | 
| -p- | Dient zum Scannen aller Ports|
| --top-ports | Scant nur Top-Ports | 
| -O | Dient zum Bestimmen des OS mit hilfe des TCP/IP-Fingerprints zu bestimmen  | 
| -A | Kombiniert mehrere Optionen wie -O und -sV um einen Detailliertere Ausgabe zu gewährleisten | 
| -Pn | Der Host wird nicht angepingt vor dem Scan |
